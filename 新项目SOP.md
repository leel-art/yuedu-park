# 新项目 SOP — 标准创建流程

> 🚀 本文档定义了在本工作站中创建新项目的标准操作流程。
> **每执行一个新项目，必须严格遵守以下全部规则。**

---

## 1. 核心铁律

| # | 规则 | 说明 |
|---|------|------|
| 1 | **独立文件夹** | 每个项目必须拥有独立的项目文件夹，禁止与其他项目混放 |
| 2 | **命名规范** | 文件夹命名必须遵循 `project-{编号}-{项目名称}` 格式 |
| 3 | **内容集中** | 所有与该项目相关的内容（代码、文档、资源、配置）全部存入对应文件夹 |
| 4 | **禁止散落** | 严禁将项目相关文件存放到电脑的其他任意位置 |

---

## 2. 文件夹命名规范

### 格式

```
project-{编号}-{项目名称}
```

### 规则

- **编号**：从 `001` 开始递增，三位数字，不足补零
- **项目名称**：使用中文全称，简洁明了，不含空格和特殊符号
- **连接符**：统一使用半角连字符 `-`

### 示例

| 正确 ✅ | 错误 ❌ | 原因 |
|---------|---------|------|
| `project-001-岳渎公园小程序` | `project1-岳渎公园` | 编号未补零 |
| `project-002-后台管理系统` | `项目2-后台管理` | 前缀不是 `project-` |
| `project-003-数据分析平台` | `project-003` | 缺少项目名称 |
| `project-004-用户中心` | `project-004 用户中心` | 使用了空格而非连字符 |

---

## 3. 项目文件夹标准结构

每个项目文件夹内部建议按以下结构组织：

```
project-{编号}-{项目名称}/
├── README.md            # 项目说明（必建）
├── src/                 # 源代码
├── docs/                # 项目文档
├── assets/              # 资源文件（图片、字体等）
├── config/              # 项目配置
├── tests/               # 测试文件
└── .gitignore           # Git 忽略规则
```

> 💡 以上为推荐结构，可根据项目类型灵活调整，但 README.md 为必建文件。

---

## 4. 新项目创建检查清单

每创建一个新项目时，必须逐项确认：

- [ ] 已确认项目编号（查询现有项目编号后递增）
- [ ] 文件夹命名符合 `project-{编号}-{项目名称}` 规范
- [ ] 项目文件夹创建在 `d:\岳渎\` 根目录下
- [ ] 已创建 `README.md` 并填写项目概述
- [ ] 所有初始文件已放入对应项目文件夹内
- [ ] 无项目文件散落在电脑其他位置
- [ ] 已在「全局工作台.md」中更新项目清单

---

## 5. 禁止事项

| 禁止行为 | 后果 |
|----------|------|
| ❌ 将项目文件存放在桌面或"下载"文件夹 | 文件丢失、版本混乱 |
| ❌ 多个项目混放在同一个文件夹 | 结构混乱、难以维护 |
| ❌ 跳号命名（如 001→003） | 编号不连续，难以统计 |
| ❌ 项目文件夹放在 `d:\岳渎\` 之外 | 无法统一管理 |
| ❌ 不写 README.md | 项目信息不明，接手成本高 |

---

## 6. 项目类型适配

不同类型项目在基础结构上的追加：

- **微信小程序**：需 `app.json`、`app.js`、`project.config.json`
- **Web 前端**：需 `package.json`、`index.html`
- **Python 项目**：需 `requirements.txt` 或 `pyproject.toml`
- **纯文档项目**：可简化为仅 `README.md` + `docs/`
