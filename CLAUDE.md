## 2. 核心行为准则

### 2.1 沟通风格
- **语言**：默认使用中文回复，代码注释使用中文。
- **简洁优先**：先给结论，再展开细节。避免冗长的前置说明。
- **先问后做**：涉及破坏性操作（删除文件、重构架构、修改配置）时，先确认再执行。
- **变更最小化**：每次修改只聚焦一个目标，不夹带无关改动。

### 2.2 代码规范

#### 通用原则
- 遵循项目现有风格，不引入新的范式或命名风格。
- 变量/函数命名：驼峰命名 `camelCase`。
- 文件名：小写 + 连字符 `kebab-case`。
- 常量/配置键：下划线 `snake_case` 或项目既有风格。
- 每个函数只做一件事，不超过 50 行为佳。
- 避免魔法数字，使用命名常量。

#### 小程序特定
- 页面生命周期按标准顺序排列：`onLoad → onShow → onReady → onHide → onUnload`。
- 数据绑定变量在 `data` 中显式声明，不动态挂载。
- `setData` 只传需要更新的字段，避免全量刷新。
- WXML 模板中复杂逻辑抽取为 WXS 或提前在 JS 中计算。
- 网络请求统一走 `utils/` 中的封装，不直接调 `wx.request`。

#### Git 规范
- Commit message 使用中文，格式：`类型: 简要描述`
  - 类型包括：`功能` `修复` `优化` `重构` `样式` `文档` `配置`
  - 示例：`功能: 新增景点语音导览页`
- 一个 commit 只做一件事。
- 不提交 `appid` 等敏感信息到仓库。

### 2.3 文件操作约定
- **新建文件前**先确认目录结构合理。
- **修改文件前**先 Read 完整内容，不盲改。
- **删除文件前**先确认没有被其他模块引用。
- 不使用 `New-Item -Force` 覆盖已有文件（会清空内容），用 Write 工具或 `Set-Content`。

## 3. 项目架构约定

```
岳渎/
├── app.js              # 小程序入口
├── app.json            # 全局配置
├── app.wxss            # 全局样式
├── components/         # 公共组件
│   └── spot-card/      # 景点卡片
├── data/               # 静态数据
├── images/             # 图片资源
├── pages/              # 页面
│   ├── index/          # 首页
│   ├── spot/           # 景点详情
│   └── video/          # 视频导览
├── utils/              # 工具函数
├── index.html          # Web 版首页（GitHub Pages）
├── project.config.json # 小程序项目配置
└── sitemap.json        # 站点地图
```

- 新增页面在 `pages/` 下新建目录，并在 `app.json` → `pages` 中注册。
- 可复用 UI 片段抽取为组件，放在 `components/` 下。
- 静态数据放在 `data/` 目录。
- 通用工具函数放在 `utils/` 目录。

## 4. 质量要求

- **改动后验证**：非 trivial 修改完成后，运行 `/verify` 确保功能正常。
- **提交前自查**：
  - 是否有 `console.log` 残留
  - 是否有硬编码的测试数据
  - 是否有未使用的变量或导入
  - 图片/资源路径是否正确
- **错误处理**：所有网络请求和用户操作必须有错误处理（try-catch / fail 回调）。
- **兼容性**：注意基础库版本 `2.33.0` 的 API 可用性。

## 5. 交互偏好

| 场景 | 行为 |
|------|------|
| 用户提出模糊需求 | 先追问澄清，不要猜测后直接执行 |
| 涉及多个方案的决策 | 列出方案对比（含利弊），给出推荐 |
| 简单 bug 修复 | 直接修复，不需要确认 |
| 架构级改动 | 先出方案（可用 `/plan`），确认后再改 |
| 不确定的技术细节 | 先查文档或源码，不要凭记忆下结论 |
| 生成大量重复代码 | 寻找抽象和复用的机会，主动建议 |

## 6. 工具使用规范

- **读文件**：使用 Read 工具，不要用 `cat`/`Get-Content`。
- **搜索内容**：使用 Grep，不要用 `Select-String`。
- **搜索文件**：使用 Glob，不要用 `Get-ChildItem -Recurse`。
- **编辑文件**：小改动用 Edit，整文件重写用 Write。
- **终端命令**：使用 PowerShell 执行，遵循其语法（参考下方速查）。

### PowerShell 速查（Windows 5.1）
```powershell
# 链式调用（无 && / ||）
command1; if ($?) { command2 }

# 文件操作须用专用工具，此处仅列例外
# 环境变量：$env:NAME = "value"
# 空值判断：$null -eq $var
# 三目替代：if ($cond) { $a } else { $b }
```
