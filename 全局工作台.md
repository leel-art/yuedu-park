# 全局工作台 — 文档体系总索引

> 📋 本文档是整个工作站文档体系的**总入口**和**使用指南**。
>
> 所有命名规范、文档使用规则、目录结构均汇总于此。

## 一、命名规范总览

### 1.1 文件命名

| 类型 | 规范 | 示例 |
|------|------|------|
| 项目文件夹 | `project-{编号}-{项目名称}` | `project-001-岳渎公园小程序` |
| 全局文档 | 中文全称 `.md` | `全局工作台.md` |
| 代码文件（小程序） | 语义化英文 `kebab-case` | `spot-card.js` |
| 代码文件（通用） | 驼峰 `camelCase`（JS）/ `kebab-case`（CSS） | `userService.js` / `user-card.wxss` |
| 配置文件 | 按框架约定 | `project.config.json` |

### 1.2 变量 / 常量命名

| 类型 | 规范 | 示例 |
|------|------|------|
| 变量 / 函数 | `camelCase` | `getSpotById()` |
| 常量 / 配置键 | `snake_case` 或项目既有风格 | `DEFAULT_ZOOM` |
| 组件名 | `kebab-case` | `spot-card` |

### 1.3 Git Commit 命名

```
类型: 简要描述
```

类型：`功能` `修复` `优化` `重构` `样式` `文档` `配置`

## 五、文档维护规则

1. **即时更新**：踩坑后立即写入「全局复利与踩坑日志」，不等"有空再说"
2. **命名一致**：新增全局文档使用中文全称 + `.md`，风格与本体系一致
3. **双向链接**：新建或更新文档时，在相关文档中追加交叉引用链接
4. **索引同步**：新增全局文档后，必须同步更新本文档「目录结构」章节
5. **定期回顾**：每完成一个项目里程碑，回顾并更新经验复利
